# 旅行手帳（trip-ledger）

多人共用的旅行記帳工具：拍收據自動辨識（日文 OCR + 翻譯）、手動快速記帳、
誰付的錢自動分帳、日圓即時換算台幣、行程地區自動判斷、預算進度與統計圖表。

正式站：https://trip-ledger.onrender.com
介紹頁（會先叫醒網站再開）：https://thomaschen-spec.github.io/trip-ledger/
（Render 免費方案，15 分鐘沒人用會睡著，第一次打開等約 1 分鐘是正常的，不是壞了）

## 使用教學（給第一次用的人）

1. **開一本旅行帳**：首頁輸入旅行名稱、總預算（日圓），可順便設一組「暗號」（邀請碼）。
   設了暗號後，只有知道暗號的旅伴進得來；每本手帳各有自己的暗號，互不相干。
2. **設定行程**：填「幾號到幾號在哪個地區」，收據會依消費日期自動標地點。
3. **旅伴加入**：把網址＋暗號傳給旅伴，每人第一次進來輸入暗號、選一個動物頭像，不用註冊帳密。
4. **記帳**：有收據按「📷 拍收據」自動辨識（可修改再存）；沒收據按「✍️ 記一筆」手動輸入。
   每筆可選「誰付的錢、哪些人分攤」；都不勾＝付款人自己出、不列入分帳。
5. **看帳與結算**：「📖 帳本」看預算進度與圖表；「🤝 結算」算出誰轉多少給誰就結清，可存成圖片分享。
6. **加到主畫面**：手機瀏覽器選單「加入主畫面」，之後像 App 一樣用。
7. **匯出**：帳本右上「⬇️ 匯出」下載 CSV，Excel 直接開。
8. **刪除**：單筆帳有「✏️ 修改／刪除」；整本旅行在「👥 成員」頁最下面永久刪除（需照抄旅行名稱防手滑）。

## 技術組合

- 後端：Flask
- 資料庫：Neon（雲端 Postgres，免費層）
- 部署：Render（免費層，Docker 環境）
- 收據辨識：Tesseract OCR（免費、開源，日文語言包用 `tessdata_best` 版本，準確度比預設版好很多）
- 翻譯：deep-translator（呼叫 Google 翻譯免費介面，非官方付費 API）
- 匯率：open.er-api.com 免費 API（每 6 小時更新一次快取，抓不到時只顯示日圓）
- 圖表：Chart.js

## 目前功能（v2.1，2026-07-15）

- 建立旅行帳本、設定預算；首頁是旅行卡片牆（日期、成員、花費進度一目了然）
- 設定行程（日期對應地區），收據會依消費日期自動判斷地區
- 多人記帳：每個人選自己的名字+頭像，不需要註冊帳密
- 拍收據 → 自動辨識店名/金額/稅金 → 翻成繁中 → 使用者確認/修改後才存檔
- ✍️ 手動快速記一筆（沒收據的花費：路邊攤、販賣機、小費）
- 🤝 每筆可記「誰付的、誰分攤」，結算頁自動算出「誰轉多少給誰」就結清
- ✏️ 已存的帳可以事後修改（不用刪掉重拍）
- 💱 日圓金額同時顯示台幣參考價（總額、每筆、結算轉帳都有）
- 儀表板：預算進度條、「今天可花多少」、每日花費趨勢、消費類別占比、付款方式分布、消費前十大
- 整體視覺統一為「旅行手帳」風格（紙質底、和紙膠帶、印章戳記）
- 🔒 邀請碼：建旅行時可設邀請碼，旅伴第一次進入要輸入暗號（成員頁可改）
- 📱 可「加到主畫面」當 App 用（PWA）
- ⬇️ 匯出 CSV（Excel 直接開不亂碼）
- 📤 結算結果一鍵存成圖片，直接分享到群組
- 收據照片自動壓縮再存檔（省資料庫空間）；行程段可修改/刪除
- GitHub Actions 每 10 分鐘 ping 一次（日本時間 08:00–24:00），旅行期間網站不會睡著
  （要停用：GitHub repo → Actions → Keep Render app awake → Disable workflow）

## 已知限制（老實列出，準確度沒有到「全自動」的程度）

收據辨識用的是免費的 Tesseract OCR + 規則比對（找「合計/小計」等關鍵字旁的數字當金額），
不是真的用 AI 理解收據內容。實測抓得到七八成準，遇到收據反光、拍歪、特殊字體排版時容易
抓錯或抓不到，所以辨識完一定會先給使用者看一次、手動修正後才存檔。

台幣換算用的是免費匯率 API 的參考價，和刷卡實際匯率會有小差距，當參考就好。

## 值得補的缺口（目前還沒做，按重要性排序）

1. **多人即時同步**：目前要手動重新整理才會看到別人剛新增的一筆，沒有做到不刷新自動更新
   （原本參考的貼文有這個效果，需要 WebSocket 之類的技術，投資大回報低，暫不做）。
2. **尚未用真實收據測試過**：目前只驗證過測試圖片，真實日文收據（各種店家、印刷品質）的
   辨識效果如何還不確定，準確度可能因收據差異很大。
3. **首頁未上鎖**：邀請碼保護的是旅行內頁（收據、金額明細）；首頁的旅行名稱、
   預算總額、已花總額仍是公開的。要完全隱藏需要再加一層。
4. **Notion 整合沒有做**：原貼文以 Notion 當資料庫並即時同步，本專案改用 Neon Postgres
   （使用者選擇），因此沒有「在 Notion App 直接看資料」的體驗。
5. 真手機的「加到主畫面」與「分享圖片」實機還沒按過（電腦模擬手機測過）。

## 資料庫欄位

`trips`（旅行）、`members`（成員）、`itinerary`（行程日期對應地區）、
`receipts`（收據：金額/稅金/分類/付款方式/日期/地區/付款人/OCR原文/翻譯/圖片）、
`receipt_splits`（分帳名單：這筆由哪些成員平均分攤）。
