# 旅遊記帳（trip-ledger）

多人共用的旅行記帳工具：拍收據自動辨識（日文 OCR + 翻譯）、行程地區自動判斷、預算進度與統計圖表。

正式站：https://trip-ledger.onrender.com
（Render 免費方案，15 分鐘沒人用會睡著，第一次打開等約 1 分鐘是正常的，不是壞了）

## 技術組合

- 後端：Flask
- 資料庫：Neon（雲端 Postgres，免費層）
- 部署：Render（免費層，Docker 環境）
- 收據辨識：Tesseract OCR（免費、開源，日文語言包用 `tessdata_best` 版本，準確度比預設版好很多）
- 翻譯：deep-translator（呼叫 Google 翻譯免費介面，非官方付費 API）
- 圖表：Chart.js

## 目前功能

- 建立旅行帳本、設定預算
- 設定行程（日期對應地區），收據會依消費日期自動判斷地區
- 多人記帳：每個人選自己的名字+頭像，不需要註冊帳密
- 拍收據 → 自動辨識店名/金額/稅金 → 翻成繁中 → 使用者確認/修改後才存檔
- 儀表板：預算進度條、每日花費趨勢、消費類別占比、付款方式分布、消費前十大

## 已知限制（老實列出，準確度沒有到「全自動」的程度）

收據辨識用的是免費的 Tesseract OCR + 規則比對（找「合計/小計」等關鍵字旁的數字當金額），
不是真的用 AI 理解收據內容。實測抓得到七八成準，遇到收據反光、拍歪、特殊字體排版時容易
抓錯或抓不到，所以辨識完一定會先給使用者看一次、手動修正後才存檔。

## 值得補的缺口（目前還沒做，按重要性排序）

1. **⚠️ 隱私/存取控制**：目前完全公開，沒有密碼或邀請碼。只要知道網址跟旅行編號
   （`/trip/<id>/...`，是循序數字，可被猜到），任何人都能看到收據內容跟花費明細。
   建議至少加一個簡單的旅行密碼或邀請碼機制。
2. **收據無法事後修改**：目前只能刪除重拍，沒有「編輯已存收據」的功能。
3. **喚醒提示**：跟 ledger-v2 一樣用 Render 免費方案，會睡著，但還沒像 ledger-v2 的介紹頁
   那樣加「叫醒中」的等待動畫，第一次打開容易被誤會當機。
4. **多人即時同步**：目前要手動重新整理才會看到別人剛新增的一筆，沒有做到不刷新自動更新
   （原本參考的貼文有這個效果，這次沒做，需要 WebSocket 之類的技術）。
5. **尚未用真實收據測試過**：目前只驗證過測試圖片，真實日文收據（各種店家、印刷品質）的
   辨識效果如何還不確定，準確度可能因收據差異很大。
6. 頭像選擇按鈕選中時沒有視覺上的反白提示，屬於小體驗瑕疵。
7. 金額只顯示日圓，沒有自動換算台幣。
8. 收據照片直接存進資料庫（Postgres bytea），沒有另外壓縮或用圖床服務；旅行時間長、
   收據量大時，資料庫容量可能會比預期成長快，目前免費額度應該夠一趟旅行用，但沒有實測過上限。
9. **介面只部分模仿原參考貼文**：已加「📷 拍照或選圖」大按鈕卡片與手機 App 風格的
   底部導覽列（儀表板/拍收據/行程/成員）；但貼文中更精緻的視覺設計（卡片動畫、
   頭像即時顯示在每筆消費上的細節排版、Notion 同步畫面）沒有做。
10. **Notion 整合沒有做**：原貼文以 Notion 當資料庫並即時同步，本專案改用 Neon Postgres
    （使用者選擇），因此沒有「在 Notion App 直接看資料」的體驗。

## 資料庫欄位

`trips`（旅行）、`members`（成員）、`itinerary`（行程日期對應地區）、
`receipts`（收據：金額/稅金/分類/付款方式/日期/地區/OCR原文/翻譯/圖片）。
